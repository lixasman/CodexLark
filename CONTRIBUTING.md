# Contributing

感谢你考虑为 CodexLark 做贡献。

本项目当前以 Windows + PowerShell 环境下的飞书长连接控制 Codex CLI 为核心方向。我们欢迎文档改进、测试补充、缺陷修复和小步迭代的功能增强，但希望所有改动都尽量保持可验证、可回滚、可审阅。

## 开始之前

在开始大改动前，请先通过 issue 或讨论说明你的目标，尤其是以下类型的工作：

- 新增较大功能面
- 修改管理员启动、权限边界或本地执行模型
- 修改会话恢复、任务存储格式或状态卡交互协议
- 新增依赖、升级依赖或计划改变发布方式

## 开发环境

建议本地环境：

- Windows
- PowerShell
- Node.js 24 或更新版本
- 可执行的 `codex` CLI（仅当你需要跑手工链路验证时）

## 本地开发流程

### 1. 安装依赖

```powershell
npm install
```

预期结果：安装完成，无依赖解析错误。

### 2. 构建

```powershell
npm run build
```

预期结果：TypeScript 编译通过，退出码为 0。

### 3. 运行自动化测试

```powershell
npm test
```

预期结果：所有测试通过，`fail` 为 0。

### 4. 按需运行手工验证脚本

当你改动以下区域时，建议补做手工验证：

- 飞书长连接连接/重连逻辑
- 会话恢复与接管逻辑
- 状态卡、启动器卡片与模式切换逻辑
- 管理员脚本、日志窗口与外部进程控制逻辑

相关脚本说明见 `scripts/selftest/README.md`。

## 贡献原则

- 保持改动聚焦：一个 PR 只解决一类问题
- 优先补测试：行为改动应尽量附带自动化测试
- 不要提交真实密钥、日志、截图、会话注册表或本地图片产物
- 不要顺手清理与本任务无关的大量文件
- 新增环境变量时，请同步更新 `README.md` 与 `.env.example`
- 修改权限模型、日志落盘或敏感数据处理逻辑时，请同步更新 `SECURITY.md`

## 依赖与发布相关变更

请谨慎处理以下改动，并在 PR 描述中单独说明原因：

- `package.json`
- `package-lock.json`
- 管理员启动脚本
- GitHub Actions
- 任何会改变公开运行方式的配置

## Pull Request 建议格式

建议在 PR 描述中包含：

- 变更目的
- 主要改动点
- 风险点
- 本地验证命令
- 如有 UI/卡片变化，附带截图或日志片段

一个合格的 PR 至少应说明：

- 改了哪些文件
- 为什么要这样改
- 如何验证
- 还有哪些已知限制

## Issue 建议

提交 issue 时，尽量提供：

- 复现步骤
- 预期行为
- 实际行为
- 运行环境（Windows 版本、PowerShell、Node.js 版本）
- 是否使用管理员权限启动
- 是否开启了调试日志
- 已脱敏的错误日志片段

## 行为准则

参与社区讨论与协作时，请遵守 `CODE_OF_CONDUCT.md`。

## 安全问题

如果你怀疑发现了安全问题，请不要公开贴出利用细节或真实凭证。请先阅读 `SECURITY.md` 中的私下披露建议。
