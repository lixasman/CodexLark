# Docs Index

本目录只保留面向公开仓库使用者的补充文档入口。

这里不再索引内部设计草案、研究记录或执行计划；如果你是第一次来到这个项目，建议先从仓库根目录的说明开始。

## 第一次来到这里，先看这些

- `README.md`：中文主说明，覆盖项目定位、环境要求、安装路径、启动方式、配置、FAQ 与已知限制
- `README.en.md`：英文概览，适合先快速了解项目范围
- `SECURITY.md`：安全披露方式、敏感区域与公开反馈注意事项
- `CONTRIBUTING.md`：贡献方式、本地验证流程与 PR 期望

## 按目标选择阅读路径

### 我想先判断这个项目适不适合我

- 先看 `README.md` 中的“适合谁”“不适合谁”“当前限制”“已知问题与注意事项”
- 如果你需要跨平台、无管理员权限或团队共享部署，可以先看限制再决定是否继续评估

### 我想尽快在本机跑起来

- 先看 `README.md` 中的“环境要求”和“飞书开放平台最小配置”
- 如果你是第一次接触这个仓库，优先走“推荐首启安装器”
- 如果你更偏向手工安装、排障或二次开发，再看“快速开始（手动路径 / 高级用户 / 排障）”

### 我想先做低风险评估，不急着接真实飞书

- 从 `README.md` 中的“初次评估路径”开始
- 先完成构建、doctor 和自动化测试，再决定是否继续接入真实飞书链路

### 我想做手工验证或深入排障

- 查看 `scripts/selftest/README.md`
- 这里收录离线验证、恢复链路验证和 app session 相关自测脚本
- 运行产物通常会写入 `artifacts/` 相关目录；在共享日志前请先自行脱敏

### 我想了解安全边界

- 查看 `SECURITY.md`
- 在公开 issue、截图、录屏或日志片段中，避免暴露真实密钥、完整日志、会话注册表和本地路径细节

## 维护原则

- `docs/` 只保留公开可维护、对使用者有帮助的文档入口
- 内部设计过程、研究材料和执行计划不在公开 `docs/` 索引中维护
- 如果未来新增公开文档，优先补充安装、排障、安全和运行边界相关内容
