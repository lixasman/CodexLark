# Security Policy

感谢你帮助 CodexLark 变得更安全。

这个项目的职责边界比较敏感：它会接收飞书事件、在本地机器上恢复/驱动 Codex CLI 会话，并且当前推荐通过管理员权限启动飞书长连接进程。因此，我们希望安全问题优先走私下披露渠道，而不是在公开 issue 中直接公开细节。

## 支持范围

当前只承诺关注最新主线代码的安全问题：

| 版本/分支 | 支持状态 |
| --- | --- |
| `main` 或默认分支最新提交 | 支持 |
| 历史提交、个人分支、长期未同步 fork | 不保证 |

## 如何报告安全问题

请优先使用以下方式之一：

1. 如果仓库启用了 GitHub Private Vulnerability Reporting，请直接通过私密安全报告提交
2. 如果仓库尚未启用私密报告，请通过维护者在仓库主页或个人资料中公开提供的私下联系方式联系
3. 如果你暂时只能提公开 issue，请只描述影响范围与现象，不要包含可复现利用细节、真实密钥、日志全文或敏感文件内容

## 报告时请包含

- 问题类型与影响范围
- 复现前提
- 风险等级判断
- 最小复现步骤
- 已做过的缓解尝试
- 是否涉及管理员权限、外部命令执行、日志泄漏或会话恢复逻辑

## 你不应该公开披露的内容

- `FEISHU_APP_SECRET` 等真实凭证
- 本地机器目录结构、用户名、工作路径等敏感环境信息
- `logs/communicate/` 下的完整日志与注册表
- `artifacts/feishu-realtest/` 或 `artifacts/selftest/` 中可能含有敏感输出的原始文件
- 飞书图片落盘目录中的真实业务图片

## 敏感区域提醒

以下区域发生问题时，请优先按安全问题处理：

- `run-admin-task.ps1` 中的提权与进程管理逻辑
- 飞书消息到本地命令/会话输入的映射链路
- 会话恢复、接管、任务日志与注册表落盘逻辑
- 调试日志、原始事件落盘与图片文件保存逻辑
- 任何可能扩大本地文件访问、命令执行或权限边界的改动

## 使用建议

如果你是普通使用者，建议：

- 仅在你信任的 Windows 机器上使用本项目
- 优先在隔离的工作目录或测试仓库中验证新行为
- 定期轮换飞书应用凭证
- 在公开演示、录屏或共享日志前先做脱敏
- 阅读并审查 `run-admin-task.ps1` 后再执行

## 响应期望

维护者会尽量在收到有效报告后尽快确认问题并评估修复优先级，但当前项目仍处于早期阶段，不承诺固定 SLA。

感谢你的负责任披露。
